background and background-image CSS properties

First Lesson: background != full component

After following the W3Schools tutorial, I found that my example was not working. At all. What I then realized was that the background image doesn't count as an html element, and since I hadn't set a default height or provided any additional content for the foreground, the browser was essentially rendering an empty div component. This was fixed by specifying a background-image or background height until content was added to define the height dynamically.

Second Lesson: A few simple observations

  • background-repeat default value is repeat.
  • background-size works similarly to no-repeat, and seems to negate repeat

Third Lesson: use display to avoid positioning struggles

Each object has the display property which in turn has the default value: 'block'. If the display value for background or background-image isn't changed, then the 'block' value will cause any float items to render outside of the background div area instead of over it. Using inline-block, on the other hand, allows for this super-imposed behavior.

Fourth Lesson: fun to play with but aesthetically challenging

While it has been fun playing with this property, it also presents a greater aesthetic challenge. All the images and colors worked out well, but adding text that was readable and still looks nice presented more of a challenge.
